# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

At Visernic Limited, we take security seriously. If you discover a security vulnerability within **ViserFly**, please follow these steps:

1.  **Do NOT** open a public issue on GitHub.
2.  Email our security team at `security@visernic.com` (or your contact email).
3.  Include a detailed description of the vulnerability and steps to reproduce it.

We will acknowledge your report within 48 hours and work on a fix immediately.

Thank you for helping keep ViserFly safe!
